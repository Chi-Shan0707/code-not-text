Key Insight

The same feature family produces dramatically different results across domains. This isn't about model capacity or data quality — it's about measurement invariance failure. The features measure different underlying constructs in each domain.

🧪 Five Methods, One Convergent Result

All five independent approaches converge on the same conclusion for coding: hand-crafted CoT-surface features don't predict code correctness.

🎯 Why This Matters

Measurement Theory

⚡

Practical Impact

Shows where cheap verifiers work (math) and where they fail (coding)—guides tool development

🎯

Domain Awareness

⚙️ Research Details

Science Data: GPQA (12,672 runs)
Coding Data: LiveCodeBench-v5 (10,688 runs)
Features: 5 hand-crafted CoT-surface features
Total Samples: 31,040 runs across all domains

📝 Precise Claims

✓ We Show

  • Hand-crafted CoT features work well in math
  • They have limited utility in science
  • They fail for coding correctness
  • This is a measurement failure, not modeling

✗ We Don't Claim

  • All text-based verifiers must fail on coding
  • Execution-aware methods won't work
  • LLM judges or raw-text classifiers are useless
  • Hidden-state probes can't capture correctness
